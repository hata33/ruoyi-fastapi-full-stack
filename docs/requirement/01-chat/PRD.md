## 1. 项目背景与目标

### 1.1 背景
随着大语言模型的普及，用户需要一个轻量、高效、易用的对话界面来与 AI 进行交互。本项目致力于复刻 DeepSeek 网页版的核心体验，提供完整的模型切换、深度推理、多模态交互能力，并保证对话交互的实时性与内容渲染的丰富性。

### 1.2 产品目标
- 提供完整的 DeepSeek 模型体验（chat/reasoner 切换、推理过程可视化）
- 实现流畅的流式对话，支持 Markdown 和代码高亮，提升阅读体验
- 构建可扩展的架构，为后续高级功能（如语音、团队协作）打下基础

## 2. 目标用户画像

- **普通用户**：日常使用 AI 辅助学习、写作、答疑，关注对话的连续性及模型选择的便捷性
- **知识工作者**：需要频繁回溯对话，对会话的标签、导出、长上下文有较高要求
- **开发者/技术爱好者**：会测试复杂代码生成，对代码高亮、重新生成、模型参数调整有强需求

## 4. 详细功能描述

### 4.1 模型管理模块

#### 4.1.1 模型选择器
- **用户故事**：根据不同场景选择合适的 AI 模型
- **触发方式**：输入框上方或设置区域的下拉选择器
- **行为逻辑**：
  - 支持切换：DeepSeek-chat（日常对话）、DeepSeek-reasoner（复杂推理）
  - 默认记住上次选择，切换时提示当前模型
  - 切换模型不影响当前会话历史

#### 4.1.2 深度思考模式
- **用户故事**：查看 AI 的推理过程，理解答案来源
- **行为逻辑**：
  - 使用 reasoner 模型时，在回复前显示"思考过程"区域
  - 思考过程默认可折叠，支持一键复制
  - 流式输出时先展示思考过程，再展示最终答案

#### 4.1.3 模型参数配置
- **用户故事**：高级用户需要精细控制模型行为
- **触发方式**：设置面板
- **行为逻辑**：
  - 支持调整：temperature（创意度）、max_tokens（长度）、top_p
  - 提供预设模板：创意型/平衡型/精确型
  - 参数按会话记忆，可随时调整

### 4.2 会话管理模块

#### 4.2.1 会话列表
- **用户故事**：快速查看并定位历史会话
- **触发方式**：左侧边栏展示
- **行为逻辑**：
  - 按更新时间倒序排列，置顶会话始终置顶
  - 显示：置顶图标、标签、会话标题、最后消息时间
  - 点击切换会话，当前会话高亮显示
  - 支持右键/悬停操作菜单

#### 4.2.2 新建对话
- **用户故事**：开启新话题时清空当前对话
- **触发方式**：点击"+ 新建对话"按钮 或 Ctrl/Cmd + N
- **行为逻辑**：
  - 创建空白会话，标题默认"新对话"
  - 右侧区域清空，输入框获得焦点
  - 发送首条消息后，标题自动更新为首条消息前20字

#### 4.2.3 删除会话
- **用户故事**：清理不需要的对话
- **触发方式**：悬停/右键菜单点击"删除"
- **行为逻辑**：
  - 二次确认后删除
  - 若删除当前会话，右侧清空或切换到最近会话
  - 删除失败需回滚并提示

#### 4.2.4 重命名会话
- **用户故事**：自定义有意义的会话名称
- **触发方式**：悬停/右键菜单选择"重命名"
- **行为逻辑**：
  - 输入框预填当前标题，限制50字
  - 回车或失去焦点保存，ESC取消
  - 空标题则恢复为默认（首条消息前20字）

#### 4.2.5 置顶/取消置顶
- **用户故事**：重要会话快速访问
- **触发方式**：悬停/右键菜单选择"置顶"/"取消置顶"
- **行为逻辑**：
  - 置顶会话按置顶时间倒序排在列表顶部
  - 显示视觉标识（如小旗图标）
  - 支持多会话同时置顶

#### 4.2.6 会话缩略
- **用户故事**：标题过长时保持列表整洁
- **行为逻辑**：
  - 标题超出宽度用省略号截断
  - 鼠标悬停显示完整标题

#### 4.2.7 会话标签/分组
- **用户故事**：用标签组织和筛选会话
- **触发方式**：会话右键菜单"添加标签"
- **行为逻辑**：
  - 支持自定义标签（如 #工作、#学习、#代码）
  - 侧边栏顶部添加标签过滤器
  - 一个会话可添加多个标签

#### 4.2.8 会话导出
- **用户故事**：将会话保存到本地或分享
- **触发方式**：会话操作菜单"导出"
- **行为逻辑**：
  - 支持导出格式：Markdown、PDF、纯文本
  - 前端生成文件下载
  - 支持批量选择导出

### 4.3 聊天交互模块

#### 4.3.1 流式消息输出
- **用户故事**：感知 AI 实时生成过程
- **行为逻辑**：
  - 用户发送消息后立即显示，AI 消息显示加载占位
  - 后端通过 SSE/WebSocket 推送文本片段，前端逐块追加
  - 支持网络中断重连
  - 流式输出期间发送按钮变为"停止"按钮

#### 4.3.2 停止生成
- **用户故事**：不需要完整回复时节省等待时间
- **触发方式**：流式输出期间点击"停止"按钮
- **行为逻辑**：
  - 立即关闭 SSE 连接，停止生成
  - 已生成内容保留在对话中
  - 停止后可重新生成或继续对话

#### 4.3.3 Markdown 渲染
- **用户故事**：正确展示格式化内容
- **行为逻辑**：
  - 支持 GFM 标准语法：标题、列表、表格、引用、粗体、斜体、代码、链接等
  - 渐进式渲染，流式输出时实时解析
  - 图片仅支持 base64 或白名单域名

#### 4.3.4 代码高亮
- **用户故事**：代码带语法高亮并可一键复制
- **行为逻辑**：
  - 识别代码块语言属性，应用对应高亮主题
  - 代码块右上角提供"复制"按钮

#### 4.3.5 消息复制
- **用户故事**：将 AI 答案复制到其他地方
- **触发方式**：AI 消息下方操作栏点击"复制"
- **行为逻辑**：
  - 复制纯文本（去除 Markdown 标记）
  - 复制成功有提示

#### 4.3.6 重新生成
- **用户故事**：对 AI 回复不满意时重新生成
- **触发方式**：已完成的 AI 消息操作栏点击"重新生成"
- **行为逻辑**：
  - 清空原消息，重新流式加载
  - 基于相同用户消息生成新回复
  - 生成期间禁用重复点击

#### 4.3.7 上下文窗口指示器
- **用户故事**：了解当前会话的 token 使用情况
- **行为逻辑**：
  - 聊天区域顶部显示进度条（如 "已用 12K / 64K tokens"）
  - 接近阈值时变色警告
  - 帮助用户判断是否需要新建会话

### 4.4 多模态能力

#### 4.4.1 文件上传
- **用户故事**：上传文档让 AI 基于内容回答
- **触发方式**：输入框区域文件上传按钮或拖拽
- **行为逻辑**：
  - 支持 PDF、Word、Excel、PPT、图片等格式
  - 显示文件预览和移除选项
  - AI 分析文件内容后回答相关问题

#### 4.4.2 联网搜索开关
- **用户故事**：获取最新网络信息
- **触发方式**：输入框附近搜索开关
- **行为逻辑**：
  - 开启后 AI 可联网搜索实时信息
  - 显示搜索来源引用
  - 默认关闭，用户主动开启

### 4.5 辅助功能

#### 4.5.1 深色模式
- **用户故事**：夜间使用保护眼睛
- **触发方式**：设置菜单切换（浅色/深色/跟随系统）
- **行为逻辑**：
  - 默认跟随系统
  - 切换后立即应用对应主题 CSS 变量

#### 4.5.2 快捷键支持
- **用户故事**：提升操作效率
- **行为逻辑**：
  - Ctrl/Cmd + Enter：发送消息
  - Ctrl/Cmd + N：新建对话
  - Ctrl/Cmd + /：查看快捷键列表
  - 设置中提供完整快捷键查看面板

#### 4.5.3 输入状态持久化
- **用户故事**：防止意外丢失已输入内容
- **行为逻辑**：
  - 使用 localStorage 按会话 ID 存储草稿
  - 页面刷新或切换会话后自动恢复
  - 发送后自动清除草稿

#### 4.5.4 欢迎页引导
- **用户故事**：新用户快速了解功能
- **行为逻辑**：
  - 首次进入或无会话时显示欢迎页
  - 展示功能介绍和"试试这些问题"快捷入口
  - 降低新用户使用门槛

#### 4.5.5 响应式布局
- **用户故事**：移动端自适应
- **行为逻辑**：
  - 宽度 < 768px 时，侧边栏默认隐藏，通过汉堡菜单呼出
  - 侧边栏呼出后选择会话或点击空白自动收起
  - 消息区域宽度自适应，字体适当调整

## 5. 可选增强功能（v2.0 规划）

根据业务发展，以下功能可作为后续版本规划：

| 功能 | 描述 | 优先级 | 价值 |
|------|------|--------|------|
| **会话搜索** | 支持标题和内容模糊搜索、高级语法 | P3 | 会话量增长后的必需功能 |
| **消息分支** | 重新生成时保留原分支，形成对话树 | P2 | 对比不同答案 |
| **语音输入/输出** | 支持语音转文字输入，AI 回复可朗读 | P2 | 提升移动端便捷性 |
| **会话分享** | 生成会话的公开分享链接（只读） | P2 | 便于传播和协作 |
| **引用回复** | 引用历史消息进行回复 | P2 | 长会话中精准追问 |
| **会话锁定** | 锁定敏感会话，访问需验证 | P2 | 保护隐私 |
| **数据统计面板** | 显示使用统计（会话数、消息数、Token 消耗） | P2 | 使用洞察 |

## 6. 非功能需求

### 6.1 性能要求
- 首次加载会话列表 < 1s（按需分页）
- 流式消息从发送到首字出现 < 1.5s（含模型调用延迟）
- 并发用户数：初期支持 1000 同时在线，后续可水平扩展

### 6.2 兼容性
- 浏览器：Chrome ≥ 90, Firefox ≥ 90, Safari ≥ 15, Edge ≥ 90
- 移动端：iOS Safari、Android Chrome 最新版本

### 6.3 安全性
- 所有接口必须鉴权（JWT 或 Session）
- 敏感词过滤：用户输入和 AI 输出需经过内容审核
- 文件上传：病毒扫描、格式验证、大小限制
- 数据备份：每日备份数据库

### 6.4 可维护性
- 前端代码模块化，组件化
- 后端 API 遵循 RESTful 规范，提供接口文档（Swagger/OpenAPI）
- 日志记录关键操作（如删除会话、模型切换），便于排查问题

## 7. 开放问题与待定事项

1. **用户认证方式**：是否支持仅本地存储（匿名试用）还是强制登录？若登录，第三方 OAuth 需求？
2. **数据保留策略**：会话保留多久？是否允许用户彻底删除？
3. **模型接入细节**：具体调用哪个 API？需要对接哪些参数？
4. **文件存储**：上传文件的大小限制、存储策略、过期清理？

以上待定项需与团队尽快讨论明确，以便进入开发阶段。
