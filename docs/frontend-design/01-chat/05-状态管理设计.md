# 聊天模块 - 状态管理设计

## 一、状态管理方案

### 1.1 技术选型

| 方案 | 优势 | 劣势 | 选择 |
|------|------|------|------|
| Pinia | Vue3 官方推荐、类型友好、轻量 | - | ✅ 采用 |
| Vuex 4 | 兼容性好 | Vue3 不推荐 | ❌ |
| Zustand | 轻量、简单 | 非 Vue 生态 | ❌ |

### 1.2 Store 结构

```
stores/
├── index.ts              # Store 入口
├── modules/
│   ├── app.ts           # 应用全局状态
│   ├── user.ts          # 用户状态
│   ├── conversation.ts  # 会话状态
│   ├── message.ts       # 消息状态
│   ├── model.ts         # 模型配置状态
│   ├── tag.ts           # 标签状态
│   └── settings.ts      # 用户设置状态
└── composables/
    ├── useChat.ts       # 聊天逻辑组合式函数
    ├── useSSE.ts        # SSE 连接管理
    └── useMessage.ts    # 消息操作
```

## 五、Message Store (消息状态)

### 5.1 状态定义

```typescript
interface MessageState {
  // 消息列表
  messages: Record<number, Message[]>  // conversationId -> messages

  // 加载状态
  loading: boolean
  loadingMore: boolean
  streaming: boolean

  // 分页
  hasMore: Record<number, boolean>  // conversationId -> hasMore
  beforeMessageId: Record<number, number | null>

  // 当前流式消息
  streamingMessage: Message | null

  // 错误
  error: string | null
}

interface Message {
  messageId: number
  conversationId: number
  role: 'user' | 'assistant' | 'system'
  content: string
  thinkingContent?: string
  tokensUsed?: number
  attachments: number[]
  userId: number
  createTime: string
  isStreaming?: boolean
  hasError?: boolean
}
```

### 5.2 Actions

| Action | 功能 | API |
|--------|------|-----|
| `fetchMessages(conversationId, beforeMessageId?)` | 获取消息列表 | GET /api/chat/conversations/{id}/messages |
| `sendMessage(conversationId, content, options?)` | 发送消息 | POST /api/chat/messages/stream |
| `stopGeneration(messageId)` | 停止生成 | POST /api/chat/messages/{id}/stop |
| `regenerateMessage(messageId, modelId?)` | 重新生成 | POST /api/chat/messages/{id}/regenerate |
| `deleteMessage(messageId)` | 删除消息 | - (本地操作) |
| `appendStreamingContent(content)` | 追加流式内容 | - (SSE) |
| `appendThinkingContent(content)` | 追加思考内容 | - (SSE) |
| `completeStreamingMessage(data)` | 完成流式消息 | - (SSE) |

### 5.3 Getters

| Getter | 功能 |
|--------|------|
| `getMessages(conversationId)` | 获取会话的消息列表 |
| `getCurrentMessages` | 获取当前会话的消息列表 |
| `hasMoreMessages(conversationId)` | 是否有更多消息 |
| `isStreaming` | 是否正在生成 |

## 九、状态持久化

### 9.1 持久化策略

| Store | 持久化方式 | 存储位置 |
|-------|-----------|----------|
| User | Token | localStorage |
| App | 主题、侧边栏状态 | localStorage |
| Conversation | 不持久化 | 内存 |
| Message | 不持久化 | 内存 |
| Model | 不持久化 | 内存 |
| Tag | 不持久化 | 内存 |
| Settings | 全部 | localStorage |

### 9.2 持久化配置

  key: 'user-store',

## 十、状态同步策略

### 10.1 服务端推送 (SSE)

| 事件 | 更新的 Store | 处理方式 |
|------|-------------|----------|
| `content_delta` | Message | 追加内容 |
| `message_end` | Message, Conversation | 完成消息、更新会话时间 |
| `thinking_delta` | Message | 追加思考内容 |

### 10.2 跨 Tab 同步

| 事件 | 同步方式 |
|------|----------|
| 登录/登出 | BroadcastChannel |
| 主题切换 | localStorage 监听 |
| 会话创建/删除 | BroadcastChannel |
| 消息发送 | 不同步 (各自独立) |

### 10.3 乐观更新

| 操作 | 乐观更新 | 回滚条件 |
|------|----------|----------|
| 发送消息 | 立即显示用户消息 | API 失败 |
| 创建会话 | 立即添加到列表 | API 失败 |
| 删除会话 | 立即从列表移除 | API 失败 |
| 置顶会话 | 立即移动位置 | API 失败 |

## 十一、错误状态处理

### 11.1 错误分类

| 错误类型 | Store | 处理方式 |
|----------|-------|----------|
| 网络错误 | App | 全局 Toast 提示 |
| 认证错误 | User | 跳转登录页 |
| 业务错误 | 各 Store | 局部错误提示 |
| SSE 错误 | Message | 显示重试按钮 |

### 11.2 错误边界

```typescript
// 组件级错误边界
<ErrorBoundary fallback={ErrorComponent}>
  <ChatArea />
</ErrorBoundary>

// 全局错误处理
app.config.errorHandler = (error) => {
  appStore.showToast({ type: 'error', message: error.message })
}
```
