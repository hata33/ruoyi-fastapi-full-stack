# 聊天模块 - 交互逻辑设计

## 一、SSE 事件流处理

### 1.1 SSE 事件类型映射

| 后端事件 | 前端处理 | UI 效果 |
|----------|----------|---------|
| `message_start` | 创建消息占位 | 显示"AI正在思考..." |
| `thinking_start` | 展开思考面板 | 显示"深度思考中..." |
| `thinking_delta` | 追加思考内容 | 流式展示思考过程 |
| `thinking_end` | 折叠思考面板 | 思考完成，显示"查看思考过程" |
| `content_delta` | 追加消息内容 | 打字机效果 |
| `message_end` | 完成消息 | 显示操作按钮、时间、Token数 |
| `error` | 错误处理 | 显示错误提示 |

### 1.2 流式渲染策略

```javascript
// 消息状态机
idle -> thinking -> generating -> completed
       ↓           ↓            ↓
     error       error        error
```

| 状态 | UI 表现 | 允许操作 |
|------|---------|----------|
| idle | 正常消息 | 复制、重新生成、删除 |
| thinking | 思考中 | 停止生成 |
| generating | 生成中 + 光标闪烁 | 停止生成 |
| completed | 正常消息 | 复制、重新生成、删除 |
| error | 错误提示 | 重试、删除 |

### 1.3 滚动行为

| 场景 | 滚动行为 |
|------|----------|
| 新消息到达 | 自动滚动到底部 |
| 流式生成中 | 持续跟随最新内容 |
| 用户向上滚动 | 暂停自动滚动 |
| 用户发送消息 | 立即滚动到底部 |
| 加载历史消息 | 保持当前位置 |

**判断逻辑**:
```javascript
// 距离底部超过 100px 时视为用户主动滚动
const isUserScrolling = () => {
  return scrollTop < scrollHeight - clientHeight - 100
}
```

## 二、消息发送流程

### 2.1 发送消息时序

```
用户点击发送
    │
    ├─ 1. 立即在 UI 上显示用户消息 (乐观更新)
    │
    ├─ 2. 禁用输入框、显示"停止生成"按钮
    │
    ├─ 3. 建立 SSE 连接
    │
    ├─ 4a. 正常流程:
    │      message_start -> 显示 AI 消息占位
    │      thinking_* -> 处理思考过程
    │      content_delta -> 流式渲染内容
    │      message_end -> 完成
    │
    └─ 4b. 异常流程:
         error -> 显示错误、重试按钮
         timeout -> 显示超时提示
```

### 2.2 输入框状态

| 时机 | 输入框状态 | 发送按钮 | 停止按钮 |
|------|------------|----------|----------|
| 空闲 | 可输入 | 禁用 | 隐藏 |
| 有内容 | 可输入 | 启用 | 隐藏 |
| 发送中 | 禁用 | 隐藏 | 显示 |
| 错误 | 可输入 | 启用 | 隐藏 |

### 2.3 防抖与节流

| 场景 | 策略 | 时间 |
|------|------|------|
| 输入框高度调整 | 节流 | 100ms |
| 搜索会话 | 防抖 | 300ms |
| 自动保存草稿 | 防抖 | 1000ms |
| 滚动位置检测 | 节流 | 100ms |

## 三、停止生成逻辑

### 3.1 停止生成的两种触发方式

| 触发方式 | UI 元素 | 交互 |
|----------|----------|------|
| 点击停止按钮 | 停止按钮 | 点击后调用停止 API |
| 离开页面 | window.beforeunload | 自动断开 SSE |

### 3.2 停止后的状态处理

```
点击停止
    │
    ├─ 1. 立即中断 SSE 连接
    │
    ├─ 2. 调用 POST /messages/{id}/stop API
    │
    ├─ 3. UI 更新:
    │      - 已生成的内容保留
    │      - 显示"[生成已停止]"提示
    │      - 恢复输入框状态
    │
    └─ 4. 提供"继续生成"选项 (可选)
```

### 3.3 部分消息的处理

| 情况 | 处理方式 |
|------|----------|
| 有内容但未完成 | 保留内容，标记"已停止" |
| 仅思考过程 | 隐藏思考面板，标记"已停止" |
| 完全无内容 | 删除占位消息 |

## 四、重新生成逻辑

### 4.1 重新生成触发

```
用户点击"重新生成"
    │
    ├─ 1. 获取当前用户消息 ID
    │
    ├─ 2. 调用 POST /messages/{id}/regenerate
    │
    ├─ 3. 如果有旧的 AI 回复:
    │      - 替换内容 (而非新增消息)
    │      - 保持消息 ID 不变
    │
    ├─ 4. 处理 SSE 流 (同发送消息)
    │
    └─ 5. 允许更换模型 (可选参数 modelId)
```

### 4.2 重新生成的 UI 表现

| 状态 | UI 变化 |
|------|---------|
| 点击时 | 消息内容淡出、显示加载中 |
| 生成中 | 流式内容覆盖旧内容 |
| 完成 | 显示新内容、更新时间 |
| 错误 | 显示错误、保留旧内容 |

### 4.3 与新消息的区别

| 项目 | 新消息 | 重新生成 |
|------|--------|----------|
| 创建新消息 | 是 | 否 |
| 更新会话时间 | 是 | 否 |
| 消息计数 | +1 | 不变 |
| 历史记录 | 包含旧 AI 回复 | 不包含旧 AI 回复 |

## 五、消息列表加载

### 5.1 初始加载

```
进入会话
    │
    ├─ 1. 加载最近 50 条消息
    │
    ├─ 2. 滚动到最底部
    │
    └─ 3. 如果有更多，显示"加载更多"提示
```

### 5.2 滚动加载 (分页)

```
用户滚动到顶部
    │
    ├─ 1. 检测: scrollTop < 100
    │
    ├─ 2. 显示加载指示器
    │
    ├─ 3. 调用 GET /conversations/{id}/messages?before_message_id={}&page_size=50
    │
    ├─ 4. 新消息插入到列表顶部
    │
    └─ 5. 保持滚动位置 (不跳转)
```

### 5.3 加载状态

| 状态 | UI 表现 |
|------|---------|
| 首次加载 | 全屏骨架屏 |
| 加载更多 | 顶部显示 Spinner |
| 加载完成 | 移除 Spinner |
| 无更多数据 | 显示"已加载全部消息" |
| 加载失败 | 显示"加载失败，点击重试" |

## 六、会话切换逻辑

### 6.1 切换会话流程

```
用户点击会话项
    │
    ├─ 1. 检查是否有未发送的草稿
    │      └─ 有: 提示保存或丢弃
    │
    ├─ 2. 中断当前 SSE 连接
    │
    ├─ 3. 清空消息列表
    │
    ├─ 4. 加载新会话消息
    │
    ├─ 5. 更新 URL (conversationId)
    │
    └─ 6. 移动端: 收起侧边栏
```

### 6.2 新建会话流程

```
用户点击"新建会话"
    │
    ├─ 1. 检查当前会话是否有消息
    │      └─ 有: 保持当前会话，清空输入框
    │      └─ 无: 调用 API 创建新会话
    │
    ├─ 2. 重置输入状态
    │
    ├─ 3. 显示欢迎界面
    │
    └─ 4. 首条消息发送后更新会话标题
```

### 6.3 会话列表更新

| 触发场景 | 更新方式 |
|----------|----------|
| 发送消息 | 当前会话移到顶部 |
| 接收消息 | 当前会话移到顶部 |
| 删除会话 | 从列表移除 |
| 置顶会话 | 移到置顶区域 |
| 重命名 | 更新标题 |

## 七、错误处理

### 7.1 网络错误

| 错误类型 | 处理方式 |
|----------|----------|
| SSE 连接断开 | 自动重试 3 次，间隔 1s/2s/5s |
| 发送失败 | 保留输入内容，显示错误提示 |
| 加载失败 | 显示重试按钮 |
| API 超时 | 显示超时提示，提供重试 |

### 7.2 业务错误

| 错误场景 | 处理方式 |
|----------|----------|
| 会话不存在 | 跳转会话列表 |
| 无权限访问 | Toast 提示，跳转会话列表 |
| 消息不存在 | 移除该消息 |
| Token 超限 | 提示升级或切换模型 |

### 7.3 错误提示规范

```
┌─────────────────────────────────────────┐
│  ⚠️  消息发送失败                        │
│  网络连接异常，请检查网络设置            │
│              [重试] [关闭]               │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  ⚠️  AI 生成失败                         │
│  服务暂时不可用，请稍后重试              │
│              [重新生成] [关闭]           │
└─────────────────────────────────────────┘
```

## 八、快捷键交互

### 8.1 全局快捷键

| 快捷键 | 功能 | 备注 |
|--------|------|------|
| Ctrl/Cmd + K | 聚焦搜索框 | - |
| Ctrl/Cmd + N | 新建会话 | - |
| Ctrl/Cmd + / | 打开快捷键面板 | - |

### 8.2 聊天快捷键

| 快捷键 | 功能 | 条件 |
|--------|------|------|
| Enter | 发送消息 | 输入框聚焦 |
| Shift + Enter | 换行 | 输入框聚焦 |
| Ctrl/Cmd + Enter | 发送消息 | 输入框聚焦 |
| Esc | 停止生成 | 生成中 |
| ↑ | 编辑上一条消息 | 仅用户消息、空闲时 |
| Ctrl/Cmd + ↑ | 上一条会话 | 输入框聚焦 |
| Ctrl/Cmd + ↓ | 下一条会话 | 输入框聚焦 |
