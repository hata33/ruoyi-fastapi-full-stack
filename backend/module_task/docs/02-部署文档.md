# 每日任务管理系统 - 部署文档

## 1. 模块概述

每日任务管理系统是一个支持每日自动刷新、完成统计、拖拽排序、禁用冻结等功能的全功能任务管理模块。

### 核心功能

| 功能 | 描述 |
|------|------|
| 每日任务 | 每天 00:00 自动重置已完成的任务 |
| 一次性任务 | 完成后永久完成，不参与刷新 |
| 长期任务 | 手动管理，不自动刷新 |
| 拖拽排序 | 支持拖拽改变任务顺序 |
| 置顶/置底 | 快速移动任务位置 |
| 禁用/启用 | 冻结任务，不参与每日刷新 |
| 完成统计 | 自动记录累计完成次数 |

## 4. API 接口

### 4.1 任务管理接口

| 方法 | 路径 | 描述 | 权限 |
|------|------|------|------|
| GET | /daily-task/list | 获取任务列表 | daily:task:list |
| GET | /daily-task/{task_id} | 获取任务详情 | daily:task:query |
| POST | /daily-task | 创建任务 | daily:task:add |
| PUT | /daily-task | 更新任务 | daily:task:edit |
| DELETE | /daily-task/{task_ids} | 删除任务 | daily:task:remove |
| PATCH | /daily-task/{task_id}/complete | 完成任务 | daily:task:edit |
| PATCH | /daily-task/{task_id}/reopen | 重开任务 | daily:task:edit |
| PATCH | /daily-task/{task_id}/disable | 禁用任务 | daily:task:edit |
| PATCH | /daily-task/{task_id}/enable | 启用任务 | daily:task:edit |
| PUT | /daily-task/reorder | 批量排序 | daily:task:edit |
| PUT | /daily-task/{task_id}/pin | 置顶/取消置顶 | daily:task:edit |

### 4.2 分类管理接口

| 方法 | 路径 | 描述 | 权限 |
|------|------|------|------|
| GET | /daily-task-category/list | 获取分类列表 | daily:category:list |

| PUT | /daily-task-category | 更新分类 | daily:category:edit |
| DELETE | /daily-task-category/{category_ids} | 删除分类 | daily:category:remove |

## 6. 权限配置

需要在系统菜单管理中添加以下权限：

### 任务管理权限
- `daily:task:list` - 查询任务列表
- `daily:task:query` - 查询任务详情
- `daily:task:add` - 新增任务
- `daily:task:edit` - 编辑任务
- `daily:task:remove` - 删除任务

### 分类管理权限
- `daily:category:list` - 查询分类列表
- `daily:category:add` - 新增分类
- `daily:category:edit` - 编辑分类
- `daily:category:remove` - 删除分类

## 7. 使用示例

### 7.1 创建每日任务

```bash
POST /daily-task
Content-Type: application/json
Authorization: Bearer <token>

{
  "title": "早起打卡",
  "description": "每天早上 6 点起床",
  "taskType": "daily",
  "iconType": "calendar",
  "categoryId": 1
}
```

### 7.2 完成任务

```bash
PATCH /daily-task/1/complete
Authorization: Bearer <token>
```

### 7.3 禁用任务

```bash
PATCH /daily-task/1/disable
Authorization: Bearer <token>
```

### 7.4 批量排序

```bash
PUT /daily-task/reorder
Content-Type: application/json
Authorization: Bearer <token>

{
  "taskOrders": [
    {"taskId": 1, "sortOrder": 1},
    {"taskId": 2, "sortOrder": 2},
    {"taskId": 3, "sortOrder": 3}
  ]
}
```

## 8. 定时任务

### 每日刷新调度器

- **执行时间**: 每日 00:00
- **功能**: 将已完成的每日任务重置为待完成状态
- **保留**: 完成次数统计 (completion_count)

### 手动测试刷新

```bash
cd backend
python -m module_task.scheduler.test_daily_refresh
```

## 9. 技术栈

- **框架**: FastAPI
- **ORM**: SQLAlchemy 2.0 (异步)
- **验证**: Pydantic v2
- **调度**: APScheduler
- **数据库**: PostgreSQL / MySQL
- **认证**: JWT + Redis

## 10. 故障排查

### 10.1 表未创建

检查 `config/get_db.py` 中的表创建逻辑是否包含新模块。

### 10.2 调度器未执行

1. 检查 `config/get_scheduler.py` 是否导入了调度器注册函数
2. 查看日志确认调度器是否注册成功

### 10.3 权限错误

在系统管理 > 菜单管理中添加对应的权限标识。
