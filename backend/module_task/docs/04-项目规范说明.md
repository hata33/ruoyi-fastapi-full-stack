# 每日任务管理系统 - 项目规范说明

## 1. 关于 `__init__.py` 文件

### 为什么不能删除 `__init__.py`？

**`__init__.py` 是 Python 包的必需文件，不能删除！**

#### 原因

1. **标识包目录**：Python 使用 `__init__.py` 来识别一个目录是否为 Python 包
2. **模块导入**：没有 `__init__.py`，Python 无法导入该目录下的模块
3. **包初始化**：`__init__.py` 可以包含包级别的初始化代码

#### 示例

```
module_task/
├── __init__.py          # 必需！标识 module_task 为 Python 包
├── service/
│   ├── __init__.py      # 必需！标识 service 为 Python 包
│   └── daily_task_service.py
```

如果没有 `__init__.py`，以下导入会失败：
```python
from module_task.service import DailyTaskService  # 会报错！
```

#### 结论

**请不要删除 `__init__.py` 文件**，它们是 Python 项目的核心组成部分。

## 2. 关于 `__pycache__` 目录

### `__pycache__` 是什么？

- `__pycache__` 是 Python 自动生成的**字节码缓存目录**
- 存储编译后的 `.pyc` 文件，用于**加速程序启动**
- 可以安全删除，Python 会自动重新生成

### 如何管理 `__pycache__`？

#### 自动忽略

已在项目中创建 `.gitignore` 文件，自动忽略缓存文件：

```gitignore
# Python cache files
__pycache__/
*.py[cod]
*$py.class
```

#### 手动清理

运行清理脚本：

```bash
cd backend
python clean_cache.py
```

## 3. 项目启动与数据库建表

### 3.1 自动建表机制

项目启动时会**自动创建数据库表**，无需手动执行 SQL 脚本。

#### 建表流程

```
启动 app.py
    ↓
调用 init_create_table()
    ↓
导入所有模块的 DO 模型
    ↓
执行 Base.metadata.create_all()
    ↓
数据库表自动创建
```

#### 关键文件

**`config/get_db.py`**

```python
# 导入所有DO模型以确保自动建表生效
import module_todo.entity.do  # noqa: F401
import module_task.entity.do  # noqa: F401  # 新增！

async def init_create_table():
    """应用启动时初始化数据库连接和表结构"""
    async with async_engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
```

#### 重要说明

✅ **已修复**：`config/get_db.py` 现已导入 `module_task.entity.do`，新模块的表会自动创建。
