# 每日任务管理系统 - 技术设计文档

## 1. 需求概述

开发一个支持每日自动刷新、可勾选完成并自动置灰、自动累计完成次数、支持拖拽排序/置顶/置底、可设置禁用冻结（不再参与刷新）、同时兼容每日循环任务/一次性任务/长期任务的个人日常任务与习惯管理工具。

## 2. 功能规格

### 2.1 任务类型
| 类型 | 代码 | 描述 | 刷新行为 |
|------|------|------|----------|
| 每日任务 | daily | 每天自动重置未完成状态 | 每日 00:00 自动刷新 |
| 一次性任务 | once | 完成后永久完成 | 不刷新 |
| 长期任务 | long | 不自动刷新，手动管理 | 不刷新 |

### 2.2 任务状态
| 状态 | 代码 | 描述 |
|------|------|------|
| 待完成 | pending | 未完成，可勾选 |
| 已完成 | completed | 已完成，置灰显示 |
| 已禁用 | disabled | 冻结状态，不参与刷新 |

### 2.3 核心功能
- **拖拽排序**：支持拖拽改变任务顺序，按 `sort_order` 字段排序
- **置顶/置底**：快速移动到列表顶部/底部
- **禁用/启用**：禁用的任务不再参与每日刷新
- **完成统计**：记录累计完成次数 (`completion_count`)
- **每日刷新**：定时任务在每日 00:00 重置 daily 类型任务的未完成项

## 7. 技术栈

- **ORM**: SQLAlchemy 2.0 (异步)
- **验证**: Pydantic v2
- **调度**: APScheduler
- **数据库**: PostgreSQL / MySQL
- **认证**: JWT + Redis (复用现有)
