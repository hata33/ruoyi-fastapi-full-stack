# 后端接口文档生成规范

## 1. 文档用途

本文档用于**后端代码完成后**，基于实际代码生成供前端使用的接口文档。

**使用场景：**
- 后端代码已由AI生成完成
- 需要提取代码中的接口信息，生成前端友好的文档
- 确保文档与实际代码100%一致

## 2. 工作流程

```mermaid
flowchart LR
    A[后端代码生成完成] --> B[AI读取Controller代码]
    B --> C[AI读取Model代码]
    C --> D[生成接口文档]
    D --> E[前端按文档开发]
```

## 4. 信息提取规则

### 4.1 从Controller提取

| 信息项 | 提取方式 | 示例 |
|--------|----------|------|
| 接口路径 | 装饰器参数 | `@router.get('/list')` → `GET /xxx/list` |
| 请求方式 | 装饰器方法 | `@router.get` → GET |
| 权限标识 | CheckUserInterfaceAuth参数 | `CheckUserInterfaceAuth('xxx:list')` → `xxx:list` |
| 功能说明 | 函数docstring | `"""获取用户列表"""` |
| 路径参数 | 函数参数无默认值 | `def get_xxx(xxx_id: int)` |
| 查询参数 | 函数参数有默认值 | `def get_list(pageNum: int = 1)` |
| 请求体 | 类型为BaseModel的参数 | `def add(xxx: XxxModel)` |
| 响应模型 | response_model参数 | `response_model=PageResponseModel` |

### 4.2 从Model提取

| 信息项 | 提取方式 | 示例 |
|--------|----------|------|
| 字段名 | 属性名转换为camelCase | `task_name` → `taskName` |
| 字段类型 | 类型注解 | `Optional[str]` → string（可选） |
| 字段说明 | Field的description | `Field(description='任务ID')` |
| 枚举值 | Literal的值 | `Literal['a', 'b']` → a/b |
| 默认值 | Field的default | `Field(default=1)` → 1 |

### 4.3 响应类型判断

| response_model | data结构 | 说明 |
|----------------|----------|------|
| PageResponseModel | {rows: [], total: int} | 分页数据 |
| XxxModel | {字段...} | 单个对象 |
| list[XxxModel] | [{...}, {...}] | 对象列表 |
| 无 | null | 无返回数据 |

## 5. 前端对接要点

生成的文档应包含以下前端开发注意事项：

```markdown
## 前端对接要点

### 5.1 基础配置
- 接口根路径：`http://localhost:9099/api`
- 认证方式：Bearer Token
- 请求头设置：`Authorization: Bearer {token}`

### 5.2 参数命名
- 请求参数使用驼峰命名（camelCase）
- 响应数据使用驼峰命名（camelCase）

### 5.3 统一响应格式
所有接口返回统一格式：
```typescript
interface ApiResponse<T> {
  code: number;      // 状态码，200表示成功
  msg: string;       // 响应消息
  data: T;           // 响应数据
}
```

### 5.4 分页处理
- 请求参数：`pageNum`（页码）、`pageSize`（每页数量）
- 响应格式：
```typescript
interface PageResponse<T> {
  rows: T[];        // 数据列表
  total: number;    // 总记录数
}
```

### 5.5 时间格式
- 所有时间字段返回格式：`yyyy-MM-dd HH:mm:ss`
- 前端传入时间格式：`yyyy-MM-dd`

### 5.6 错误处理
```typescript
if (response.code === 200) {
  // 成功处理
} else {
  // 显示错误信息 response.msg
}
```

### 5.7 文件上传
- 使用FormData格式
- 请求头：`Content-Type: multipart/form-data`
```
