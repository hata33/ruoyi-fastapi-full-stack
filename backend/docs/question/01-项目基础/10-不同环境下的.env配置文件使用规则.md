# 不同环境下的 .env 配置文件使用规则

## 问题描述

backend 目录下存在多个 `.env` 配置文件：
- `.env.dev` - 开发环境配置
- `.env.prod` - 生产环境配置
- `.env.docker` - Docker 环境（已废弃）

不同启动方式下，使用哪个配置文件？

---

## 配置加载原理

### 核心代码位置

**`config/env.py:204-228`** - `parse_cli_args()` 方法：
```python
run_env = os.environ.get('APP_ENV', '')
env_file = '.env.dev'  # 默认加载 .env.dev
if run_env != '':
    env_file = f'.env.{run_env}'
load_dotenv(env_file)  # 默认 override=False，不覆盖已存在的环境变量
```

**`pydantic_settings.BaseSettings`** 配置读取优先级：
1. 系统环境变量（优先级最高）
2. .env 文件
3. 类默认值（优先级最低）

## 三种启动方式的配置使用

### 1. 本地开发

```bash
python backend/app.py
# 或
python backend/app.py --env=dev
```

| 配置项 | 来源 |
|--------|------|
| APP_ENV | 命令行参数（默认 dev） |
| 其他配置 | 直接使用 `backend/.env.dev` |

---

### 2. Docker 镜像单独运行

**文件：`fastapi-dockerfile`**
```dockerfile
# 第 29 行：复制生产环境配置
COPY ./backend/.env.prod .env

# 第 48 行：启动命令
CMD ["python", "app.py", "--env=prod"]
```

| 配置项 | 来源 |
|--------|------|
| APP_ENV | Dockerfile 启动命令指定为 `prod` |
| 其他配置 | 直接使用 `backend/.env.prod` |

**⚠️ 重要**：`.env.prod` 中的数据库/Redis 配置指向容器内部（如 `127.0.0.1`），这是为镜像单独运行设计的。

---

### 3. docker-compose 部署

**文件：`docker-compose.yml`**

```yaml
hata-server:
  environment:
    DB_HOST: hata-postgresql      # 覆盖 .env.prod
    DB_PORT: 5432
    DB_USERNAME: ${POSTGRESQL_ROOT_NAME}
    DB_PASSWORD: ${POSTGRESQL_ROOT_PASSWORD}
    REDIS_HOST: hata-redis        # 覆盖 .env.prod
    REDIS_PORT: 8761
    REDIS_PASSWORD: ${REDIS_PASSWORD}
```

| 配置项 | 来源 |
|--------|------|
| APP_ENV | 启动命令指定为 `prod` |
| DB_HOST/PORT | docker-compose `environment` 覆盖 |
| REDIS_HOST/PORT | docker-compose `environment` 覆盖 |
| 其他配置 | `backend/.env.prod` |

**覆盖机制**：
1. docker-compose 的 `environment` 变量先注入到容器环境
2. `load_dotenv('.env.prod')` 执行时，**不会覆盖**已存在的环境变量
3. `BaseSettings` 读取时，docker-compose 设置的变量优先级更高

## 配置文件用途对照表

| 启动方式 | 基础配置 | DB/Redis 配置 |
|---------|---------|---------------|
| 本地开发 | `.env.dev` | `.env.dev`（本地数据库） |
| Docker 镜像 | `.env.prod` | `.env.prod`（127.0.0.1，需外网 DB） |
| docker-compose | `.env.prod` | **docker-compose 覆盖**（Docker 服务名） |

## 废弃文件

### `.env.docker`

**状态**：❌ 已废弃，未被任何代码引用

**证据**：
- `fastapi-dockerfile` 使用的是 `.env.prod`
- `docker-compose.yml` 的注释只是历史遗留
- `app.py` 的 help 文本提到 `docker` 选项但未实际使用

**建议**：可删除此文件，或用于统一管理 docker-compose 的环境变量配置。
