# 两种启动方式的关系

## 问题

命令行中写了 `server:app`，在 app.py 中也写了 `server:app`，到底是哪个启动命令最终生效的？为什么这样做？

## 回答

**两种启动方式，互不干扰：**

```
┌─────────────────────────────────────────────────────────┐
│                    启动方式1                             │
├─────────────────────────────────────────────────────────┤
│  命令：uvicorn server:app --reload                       │
│                                                         │
│  命令行的 server:app 生效 ✓                              │
│  app.py 里的代码不执行 ✗                                 │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    启动方式2                             │
├─────────────────────────────────────────────────────────┤
│  命令：python app.py --env dev                           │
│                                                         │
│  进入 if __name__ == '__main__'                         │
│  app.py 代码里的 server:app 生效 ✓                       │
└─────────────────────────────────────────────────────────┘
```

**为什么写两遍？**

不是重复！是给用户**两个选择**：

| 选择 | 优点 | 缺点 |
|------|------|------|
| `uvicorn server:app` | 简单直接，uvicorn 原生方式 | 需要手动写参数 |
| `python app.py` | 可以加 `--env dev` 参数，更灵活 | 多了一层包装 |

**app.py 的作用：** uvicorn 命令的**包装器**

```python
# 你写：
python app.py --env dev

# 相当于帮你执行：
uvicorn server:app --host 0.0.0.0 --port 8000 --reload
```
