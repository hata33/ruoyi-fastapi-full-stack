# 复杂需求的开发流程

## 问题

简单需求可以直接动手，复杂需求应该怎么处理？

## 回答

### 核心原则

```mermaid
flowchart LR
    subgraph 简单需求
        A1[想一下] --> B1[直接编码]
    end

    subgraph 复杂需求
        A2[分析] --> B2[设计]
        B2 --> C2[编码]
    end

    style A2 fill:#ff6b6b
    style B2 fill:#feca57
    style C2 fill:#48dbfb
```

```mermaid
pie title 时间分配对比
    "简单需求-设计" : 10
    "简单需求-编码" : 90
    "复杂需求-设计" : 40
    "复杂需求-编码" : 60
```

## 完整流程

```mermaid
flowchart TD
    subgraph 阶段1[阶段1: 需求分析]
        A1[功能边界]
        A2[业务规则]
        A3[异常场景]
        A4[性能/安全要求]
    end

    subgraph 阶段2[阶段2: 技术设计]
        B1[技术选型]
        B2[架构设计]
        B3[核心流程设计]
        B4[边界方案]
    end

    subgraph 阶段3[阶段3: 数据库设计]
        C1[ER图]
        C2[表结构]
        C3[索引设计]
    end

    subgraph 阶段4[阶段4: 接口设计]
        D1[接口列表]
        D2[请求/响应]
        D3[权限设计]
    end

    subgraph 阶段5[阶段5: 编码实现]
        E1[Entity]
        E2[DAO]
        E3[Service]
        E4[Controller]
    end

    阶段1 --> 阶段2 --> 阶段3 --> 阶段4 --> 阶段5

    style 阶段1 fill:#ff6b6b
    style 阶段2 fill:#feca57
    style 阶段3 fill:#48dbfb
    style 阶段4 fill:#1dd1a1
    style 阶段5 fill:#5f27cd
```

## 示例：任务分配系统

### 需求分析

```mermaid
mindmap
    root((任务分配系统))
        功能
            管理员创建任务
            分配给成员
            成员接收/拒绝
            优先级&截止时间
            状态流转
            评论/附件
            转派/协作
        实体关系
            用户/角色/权限
            任务/任务成员
            任务评论/附件
            通知/日志
        边界场景
            任务被拒绝?
            负责人离职?
            多人编辑冲突?
```

### 技术设计

```mermaid
flowchart TB
    subgraph 技术选型
        T1[WebSocket<br/>实时通知]
        T2[Redis<br/>任务队列]
        T3[定时任务<br/>检查截止时间]
    end

    subgraph 架构
        L1[Controller 层]
        L2[Service 层<br/>TaskService<br/>NotificationService<br/>WorkflowService]
        L3[DAO 层]
        L4[数据库 + Redis]
    end

    L1 --> L2 --> L3 --> L4
```

### 数据库 ER 图

```mermaid
erDiagram
    USER ||--o{ TASK : creates
    USER ||--o{ TASK_USER : assigned
    TASK ||--o{ TASK_USER : has
    TASK }o--|| CATEGORY : belongs
    TASK ||--o{ COMMENT : has
    TASK ||--o{ ATTACHMENT : has

    USER {
        bigint user_id PK
        string user_name
        string role
    }

    TASK {
        bigint task_id PK
        string title
        text content
        string status
        string priority
        datetime deadline
    }

    TASK_USER {
        bigint task_id FK
        bigint user_id FK
        string role
    }

    CATEGORY {
        bigint category_id PK
        string name
    }

    COMMENT {
        bigint comment_id PK
        bigint task_id FK
        string content
    }

    ATTACHMENT {
        bigint file_id PK
        bigint task_id FK
        string file_path
    }
```

```mermaid
    style T1 fill:#48dbfb

```

## 关键区别总结

```mermaid
graph LR
    subgraph Simple[简单需求]
        S1[文档: 不需要]
        S2[设计图: 不需要]
        S3[技术方案: 不需要]
        S4[接口文档: 边写边定]
        S5[开发方式: 一个人]
        S6[测试: 简单测试]
        S7[时间: 设计10% 编码90%]
```
